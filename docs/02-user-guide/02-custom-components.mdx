# DocsKit components

The DocsKit package already includes a set of ready-made custom components that you can use in your MDX documentation pages.  

## Message component 

With this component you can display notifications or info bars in your pages.  

Error message example:

<Message variant="error">
This is an error message
</Message>

How to add it to your page:
```JSX
<Message variant="error">
  This is an error message
</Message>
```
---------

Info message example:

<Message filled>
This is a filled info message: the `filled` attribute adds a colorful background
</Message>

How to add it to your page:
```JSX
<Message variant="warning" filled>
  This is a filled warning message: the `filled` attribute adds a colorful background
</Message>
```

Variant:
- info (default)
- success
- warning
- error

## Card component 

Using the Card component you can create content teasers or highlighted content blocks in your markdown pages.

<Card shadow hoverable>
 *Card example*  
 You can add markdown to the card content
</Card>

```markdown
<Card shadow hoverable>
 *Card example*  
 You can add markdown to the card content
</Card>
```
------

<Card variant="highlighted" hoverable>
 *Highlighted card*  
 You can add markdown to the card content
</Card>

```markdown
<Card variant="highlighted" hoverable>
 *Highlighted card*  
 You can add markdown to the card content
</Card>
```
------

<Card variant="gradient">
 *Gradient card*  
 You can add markdown to the card content
</Card>

```markdown
<Card variant="gradient">
 *Gradient card*  
 You can add markdown to the card content
</Card>
```

## Button component

Cou can add styled CTA buttons using the `<Button>` component:

<Button href="#">Primary button</Button>


```JSX
<Button href="#">Primary button</Button>
```
------
<Button href="#" variant="secondary">Secondary button</Button>


```JSX
<Button href="#" variant="secondary">Secondary button</Button>
```

## Grid component

Cou can use the `<Grid>` component to create multi-column layouts.  
The layout will automatically fall back to a single-colument layout on mobile devices with smaller screens.

<Grid>
  <Card variant="highlighted" hoverable>
  Column 1
  </Card>
  <Card variant="highlighted" hoverable>
  Column 2
  </Card>
</Grid>

```JSX
<Grid>
  <Card variant="highlighted" hoverable>
  Column 1
  </Card>
  <Card variant="highlighted" hoverable>
  Column 2
  </Card>
</Grid>
```
------
<Grid cols="3">
  <Card variant="highlighted" hoverable>
  Column 1
  </Card>
  <Card variant="highlighted" hoverable>
  Column 2
  </Card>
  <Card variant="highlighted" hoverable>
  Column 3
  </Card>
</Grid>
